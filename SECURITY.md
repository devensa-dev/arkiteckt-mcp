
# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly.

**Do not open a public issue.**

Instead, email:

security@<your-domain>.com

Please include:
- A description of the issue
- Steps to reproduce (if applicable)
- Potential impact
- Any suggested mitigations

---

## Scope

This policy applies to:
- The MCP server
- CLI tooling
- Architecture schemas
- Validation and rule engines

It does not apply to:
- Third-party dependencies
- User-managed infrastructure
- Cloud provider misconfigurations outside this projectâ€™s control

---

## Disclosure Process

- We will acknowledge receipt within a reasonable timeframe
- We will investigate and address valid issues
- We will coordinate disclosure when appropriate

We do not currently run a bug bounty program.

---

## Security Philosophy

This project is designed with the following security principles:

- Least privilege
- Deterministic behavior
- Transparent enforcement
- No hidden side effects

Security issues are treated with high priority.
